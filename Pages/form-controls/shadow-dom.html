🔒 Shadow DOM Exploration

Learn about Open and Closed Shadow DOM implementations and their differences

OPEN Open Shadow DOM

Open Shadow DOM allows external scripts and styles to access and manipulate the shadow DOM structure. The DOM tree is accessible via JavaScript from outside the component.

💡 Can be inspected and modified from outside

Open Shadow DOM Example

Shadow DOM will be rendered here...

Result: 

                

            


            

                <script>
                // Create Open Shadow DOM
                const openHost = document.getElementById('open-shadow-host');
                const openShadow = openHost.attachShadow({ mode: 'open' });

                // Add styles and content
                openShadow.innerHTML = `
                <style>
                :host { display: block; }
                .shadow-content { color: #0c4a6e; background: #dbeafe; padding: 1rem; border-radius: 6px; }
                </style>
                <div class="shadow-content">
                <h4>Open Shadow DOM Content</h4>
                <input type="text" id="open-input" placeholder="This can be accessed from outside">
                </div>
                `;
                </script>
            

        


        
        

            

                CLOSED Closed Shadow DOM
            

            

                Closed Shadow DOM encapsulates the DOM completely. External scripts cannot access the shadow DOM
                structure
                unless a reference is stored within the component. Maximum encapsulation and security.
            


            
🔐 Completely hidden from external access


            

                
Closed Shadow DOM Example

                

                    
Shadow DOM will be rendered here...

                


                

                    
                    
                    
                


                

                    Result:
                    

                

            


            

                <script>
                // Create Closed Shadow DOM with stored reference
                const closedHost = document.getElementById('closed-shadow-host');
                const closedShadow = closedHost.attachShadow({ mode: 'closed' });

                // Store reference for internal use
                closedHost._shadowRoot = closedShadow;

                closedShadow.innerHTML = `
                <style>
                :host { display: block; }
                .shadow-content { color: #7c2d12; background: #fecaca; padding: 1rem; border-radius: 6px; }
                </style>
                <div class="shadow-content">
                <h4>Closed Shadow DOM Content</h4>
                <input type="text" id="closed-input" placeholder="Hidden from outside">
                </div>
                `;
                </script>
            

        


        
        

            
📊 Open vs Closed Shadow DOM

            

                

                    
                        

                            Feature
                            
                                OPEN Open Shadow
                                DOM
                            
                            
                                CLOSED Closed
                                Shadow DOM
                            
                        

                    
                    
                        

                            External Access
                            ✅ Can access via element.shadowRoot
                            ❌ Cannot access from outside
                        

                        

                            Modification
                            ✅ Can modify shadow DOM from outside
                            ❌ Cannot modify from outside
                        

                        

                            Encapsulation
                            ⚠️ Partial encapsulation
                            🔐 Full encapsulation
                        

                        

                            Security
                            ⚠️ Medium security level
                            ✅ Higher security level
                        

                        

                            Testing
                            ✅ Easier to test
                            ❌ Harder to test
                        

                        

                            Dev Tools
                            ✅ Visible in DevTools
                            ✅ Visible in DevTools
                        

                        

                            Use Case
                            Public components, libraries
                            Sensitive components, plugins
                        

                    
                

            

        


        
        

            
🎯 Key Insights

            
    
                
  • Open Shadow DOM: Use when you want component
                    flexibility and external control is acceptable.
    • 
                
  • Closed Shadow DOM: Use for sensitive operations,
                    premium features, or when you want maximum protection.
    • 
                
  • Encapsulation: Shadow DOM provides CSS and DOM
                    isolation regardless of open/closed mode.
    • 
                
  • Both are visible: Both modes are visible in browser
                    DevTools for debugging.
    • 
                
  • Note: Closed mode is not truly "secure" - it's only hidden from JavaScript, not
                    from inspection tools.
