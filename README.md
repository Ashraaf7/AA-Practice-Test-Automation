# Test Automation Practice Platform

A general automation practice application with clear topics, guided exercises, and a practical test automation learning path.

**Credentials**: username: `admin` | password: `admin`

## What this project offers

- Framework-agnostic automation practice pages.
- Simple UI focused on testability and automation scenarios.
- Hands-on coverage of locators, interactions, forms, waits, windows, dialogs, and advanced flows.
- A course overview page that maps meaningful practice topics and paths.

## Core practice areas

- **Course Overview**: Understand the learning roadmap and how to use the practice app.
- **Element Locators**: ID, CSS, XPath, selectors, and locator strategy.
- **Form Controls**: Inputs, checkboxes, radio buttons, dropdowns, date pickers, and file upload.
- **Browser Actions**: Click, double-click, right-click, keyboard, drag & drop, and scrolling.
- **Navigation & Pages**: Windows, tabs, frames, iframes, shadow DOM, page state, and results checks.
- **Dialogs & Alerts**: Alerts, confirm dialogs, prompts, modals, and validation flows.
- **Synchronization**: Timers, timeouts, waits, retries, and stable automation behavior.
- **Data & Tables**: Tables, lists, data extraction, sorting, and verification.

## How to use

1. Open `index.html` in your browser.
2. Login with `admin` / `admin`.
3. Open the main practice dashboard.
4. Choose a topic and practice writing automation tests.

## Notes for SDET practice

- The pages are intentionally programmatic and easy to inspect.
- The project does not require a specific automation framework.
- Use this app to practice selectors, actions, assertions, and page flow testing.
- Keep exercises general so they can be reused with any browser automation tool.

## Recommended pages

- `Pages/course-overview.html` вЂ” roadmap and topic categories.
- `Pages/main.html` вЂ” the practice dashboard with current exercises.
- `Pages/about.html` вЂ” project purpose and test focus.
